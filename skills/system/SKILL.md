# AI行为记录日志

## 功能
记录AI自身的重要行为和决策，让用户能够追溯AI在本次会话中做了什么重要操作。

## 日志文件
```
~/.hermes/logs/ai_actions.log
```

## 记录格式
```
[YYYY-MM-DD HH:MM] 分类 | 事件描述 | 详情（可选）
```

## 分类标签
- `系统` — 系统级事件（记忆满、工具异常、session变化）
- `记忆` — 记忆的读写操作（保存、更新、压缩）
- `工具` — 工具调用和结果（浏览器、API调用、脚本执行）
- `研究` — 研究类行为完成（源码分析、信息检索）
- `决策` — AI主动做出的判断（跳过步骤、变更计划）
- `用户` — 需要用户注意的事件（权限问题、选项汇报）
- `错误` — 操作失败或异常

## 触发条件（必须记录）
以下情况必须立即写入ai_actions.log，**不允许私下处理**：

| 场景 | 分类 | 说明 |
|------|------|------|
| MEMORY空间超过80% | 系统 | 立即通知用户，不私下压缩 |
| MEMORY保存失败 | 用户 | 报告失败原因，等用户指示 |
| Session被分割 | 系统 | 记录原session→新session |
| 发现工具异常 | 错误 | 记录异常详情 |
| 重要研究完成 | 研究 | 简要记录结论 |
| 工具调用异常 | 错误 | 记录错误信息 |

## 主动记录（鼓励记录）
- 重要研究结论完成
- 发现新的用户偏好或习惯
- 工作流程的重大变更
- 系统配置的修改

## 格式示例
```
[2026-04-26 11:30] 系统 | AI行为记录日志建立 | 用户要求建立选项B
[2026-04-26 11:35] 记忆 | MEMORY.md已压缩 | 3581→1630 chars
[2026-04-26 11:40] 研究 | ComfyUI架构分析完成 | poll_op/GraphBuilder/三层缓存
[2026-04-26 11:45] 工具 | Flask服务器启动成功 | localhost:8765
```

## 注意事项
- 记录要简洁，核心信息不丢失
- 同一会话内的连续操作可合并记录
- 用户明确要求不记录的场景除外
